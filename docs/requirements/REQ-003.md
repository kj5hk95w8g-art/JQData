# REQ-003: 日线数据全量同步

> 状态：⚠️ 进行中
> 最后更新：2026-05-09

---

## 一、一句话描述

将聚宽 JQData 沪深 A 股日线行情数据（2020-2026 年）同步到本地 ClickHouse，支持前复权和后复权两种口径。

---

## 二、背景与用户场景

**场景：** 量化策略回测、投研分析需要本地化的历史行情数据，避免每次回测都调用 JQData API（速度慢、有额度限制、网络不稳定）。

**现状：** JQData 正式账号每日约 1000 万条额度。同步任务每日上限 600 万条，预留 400 万条给其他用户使用。

---

## 三、数据范围

| 维度 | 范围 |
|------|------|
| 标的 | 沪深 A 股全市场（约 5201 只）+ 主要指数（15 个）|
| 时间 | 2020-01-01 至 2026-05-08（近 5 年+）|
| 数据类型 | 日线行情（open/high/low/close/volume/money）|
| 复权口径 | 前复权（pre）、后复权（post）各一份 |
| 不做 | 分钟线、基金、期货、财务数据 |

### 3.1 目标表

| 表名 | 内容 | 引擎 |
|------|------|------|
| `index_daily` | 指数日线 | MergeTree |
| `stock_daily_pre` | 股票前复权日线 | ReplacingMergeTree |
| `stock_daily_post` | 股票后复权日线 | ReplacingMergeTree |

表结构详见：`docs/database-schema.md`

---

## 四、同步策略

### 4.1 分批策略

| 参数 | 值 |
|------|-----|
| 每批标的数 | **200 只** |
| 每批数据量 | 约 26 万条（200 × 1300 交易日）|
| 同步任务日额度上限 | **600 万条**（预留 400 万给其他用户）|
| 日额度下可跑批次 | 约 23 批/天 |
| 全量预估批次 | pre 27 批 + post 27 批 = **54 批** |
| 全量预估时间 | **3 天（pre）+ 3 天（post）= 6 天** |

### 4.2 执行顺序

```
Phase 1: index_daily（15 个指数）-> 已完成
Phase 2: stock_daily_pre（前复权，27 批）-> 进行中（9/27 批）
Phase 3: stock_daily_post（后复权，27 批）-> 待开始
```

### 4.3 幂等设计

- 使用 `ReplacingMergeTree` 引擎，重复写入自动去重（按 code + trade_date）
- 同步脚本记录 checkpoint（Redis）到批次级别，支持断点续传
- 支持按批次重跑，不会重复累加

### 4.4 额度管理

- 每日上限 600 万条（环境变量 `DAILY_QUOTA_LIMIT` 控制）
- 每批后检查当日已用量，超过上限自动暂停并记录断点
- 次日从断点继续

## 五、定时同步（增量）

### 5.1 触发时机

交易日 **17:30**（收盘后），通过 crontab 自动触发。

### 5.2 增量逻辑

- 读取 Redis checkpoint 中各表最后同步日期
- 只同步 **上次同步日期的次日 至 当天** 的数据
- 增量数据量极小：5201 只 × 1 天 ≈ 5200 条/口径，不会触发额度限制

### 5.3 crontab 配置

```cron
# 交易日 17:30 执行增量同步
30 17 * * 1-5 cd /data/jqdata-platform && python3 src/sync_daily.py --incremental >> /data/logs/jqdata-sync.log 2>&1
```

> 全量同步（historical backfill）**不通过 crontab**，需要手动 SSH 登录执行。

## 六、验收标准

### 6.1 完整性校验

- [ ] `index_daily`：15 个指数，每个指数 1300+ 交易日，无缺失
- [ ] `stock_daily_pre`：5201 只 × 1300 交易日 ≈ 676 万条
- [ ] `stock_daily_post`：5201 只 × 1300 交易日 ≈ 676 万条
- [ ] 每只股票的交易日数量与指数交易日数量一致（排除停牌）

### 6.2 价格异常检测

- [ ] 无价格为 0 或负数的记录
- [ ] 无成交量为负数的记录
- [ ] 单日涨跌幅不超过 ±20%（ST 股 ±5%）
- [ ] 前复权价格连续性检查（无跳变）

### 6.3 性能指标

- [ ] 单批次同步时间 < 30 分钟
- [ ] 批量写入 ClickHouse 速度 > 10 万条/秒

## 七、异常处理

| 异常场景 | 处理方案 |
|---------|---------|
| JQData API 超时/失败 | 重试 3 次，间隔 5 秒；仍失败记录日志，跳过该批次后续补跑 |
| 同步任务日额度（600万）耗尽 | 记录 checkpoint，次日从断点继续 |
| ClickHouse 写入失败 | 分批降级，单批拆成更小的批次重试 |
| 数据缺失（某只股票某段日期无数据） | 记录到缺失日志，人工确认是否为退市/停牌 |
