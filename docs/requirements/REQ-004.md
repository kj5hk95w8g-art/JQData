## 一、一句话描述

公司内部版 akshare：团队成员 pip install 后开箱即用，无需配置、无感知认证，像调用 akshare 一样查询行情数据。

## 二、背景与用户场景

**场景：** 团队成员（投资部、研究部）需要在外网（家里/出差）也能查询本地化的行情数据，但不需要管理账号、申请 Token、配置 base_url。

**现状：** REQ-002 已提供 REST API + Python SDK，但用户需要手动设置 base_url 和 api_key，门槛高。

## 四、技术方案

### 4.1 认证：请求签名（简化版）

**原理：** SDK 内置盐值，每次请求自动计算签名，服务器比对。

```
SDK 内置盐值（SECRET_SALT）
    -> 每次请求：signature = md5(SALT + timestamp)
    -> 请求头携带 X-Timestamp + X-Signature
    -> 服务器验证签名（比对即可，不检查时间窗口）
```

**为什么不用明文 Token：** 签名每次不同，抓包无法直接复用。

**关键简化决策：**

| 原方案 | 简化后 | 理由 |
|--------|--------|------|
| 检查时间窗口 `|now - timestamp| <= 300秒` | **不检查** | 内部团队使用，重放攻击风险极低；避免用户电脑时间不准导致报错 |
| 每年更换盐值，多盐值并行 7 天 | **不主动更换，泄露再换** | 私有仓库泄露概率极低；减少维护工作量 |
| Redis 限流 | **Nginx limit_req** | 只改配置，不碰代码 |

### 4.3 SDK 体验

**安装：**
```bash
pip install git+ssh://git@github.com:kj5hk95w8g-art/JQData.git#subdirectory=src/sdk
```

**使用（完全无感知）：**
```python
import jqdata_sdk as jq

# 无需 auth()，内置 base_url 和签名逻辑
df = jq.get_price("000001.XSHE", start_date="2020-01-01", end_date="2026-05-08")
```

**升级：**
```bash
pip install --upgrade git+ssh://git@github.com:kj5hk95w8g-art/JQData.git#subdirectory=src/sdk
```

### 4.4 安全防护层次

| 层次 | 机制 | 作用 | 状态 |
|------|------|------|------|
| 第一层 | 请求签名 | 阻止无签名的随机扫描 | ✅ 已完成 |
| 第二层 | GitHub 私有仓库 | 降低被发现概率 | ✅ 已完成 |
| 第三层 | Nginx 单 IP 500r/s 限流 | 防止死循环拖垮服务器 | ✅ 已完成 |

### 4.5 架构

```
团队成员电脑
    Python SDK (jqdata_sdk)
        - 内置 base_url
        - 内置 SECRET_SALT（硬编码）
        - 每次请求自动附加签名头
                |
                | HTTP
                v
        Nginx（反向代理 + 单 IP 500r/s 限流）
            limit_req_zone + limit_req
            proxy_pass -> api:8000
                |
                v
        FastAPI (jqdata-api)
            /health 免检
            签名验证中间件（比对 md5(SALT+ts)）
            路由层 -> ClickHouse 查询
```

**代码/配置量：** FastAPI 中间件 ~15 行 + SDK 签名 ~10 行 + Nginx 限流 ~8 行 = **~33 行**

## 五、团队成员接入流程

### 5.1 生成 SSH key

```bash
ssh-keygen -t ed25519 -C "your_name@company.com"
# 保存位置回车默认，密码直接回车（不设置）
cat ~/.ssh/id_ed25519.pub
```

### 5.2 发邮件申请

**收件人：** 仓库管理员
**主题：** 申请 JQData SDK 权限
**正文：** 姓名、部门、邮箱、SSH 公钥内容

### 5.3 管理员开通

GitHub 仓库 Settings -> Deploy keys -> Add deploy key（只读，不勾选 write access）

## 六、风险与应对

| 风险 | 应对方案 |
|------|---------|
| **盐值泄露** | 不主动更换，发现泄露后直接切换新盐值，发新版 SDK |
| **DDoS 攻击** | Nginx 单 IP 500r/s 限流；极端情况换弹性公网 IP |
| **代码泄露** | 定期换盐值；离职移除 GitHub Deploy Key |
| **GitHub 访问失败** | 保留本地 wheel 打包作为备用分发方式 |
