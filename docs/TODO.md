### REQ-004: 轻量级数据分发平台
- **类型**: feat
- **状态**: 🟡 已明确，待开发
- **依赖**: REQ-002, REQ-003
- **描述**: 像akshare一样，公司团队成员pip install即可使用，外网也能访问

**核心原则：**
- 不做用户管理系统
- 不做付费/额度售卖
- 不做手动发Token环节
- 靠"低调+内置Token+IP限流"控制访问范围

**技术方案：**
1. **入口**：弹性公网IP绑定D服务器（101.132.161.52），不走域名，无需备案
2. **认证**：SDK内置硬编码Token，用户无感知
   - 服务器端校验Token有效性
   - Token泄露时可更换，发新版SDK即可
3. **防刷保护**：单IP限流100条/秒（Nginx层或FastAPI层实现）
4. **SDK体验**：
   ```python
   pip install jqdata-sdk
   import jqdata_sdk as jq
   # 无需auth()，开箱即用
   df = jq.get_price("000001.XSHE", start_date="2020-01-01", end_date="2026-05-08")
   ```
5. **分发范围**：GitHub仓库低调公开，不宣传、不上PyPI、不发推广帖

**不做：**
- ❌ 用户注册/登录系统
- ❌ 付费购买/积分系统
- ❌ 手动Token分发
- ❌ 每日全局额度限制（内部使用，单IP限流足够）
- ❌ 独立文档站（GitHub README足够）
- ❌ 用户管理后台

**验收标准：**
- [ ] 公司团队成员在外网（家里/出差）能pip install后直接使用
- [ ] 单IP限流生效，死循环查询不会拖垮服务器
- [ ] Token可更换，旧Token自动失效
- [ ] GitHub仓库更新后，团队成员可pip install --upgrade升级
