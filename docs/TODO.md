# JQData 需求池

> 状态：`待评审 → 待排期 → 已排期 → 开发中 → 待测试 → 已发布`
>
> 编号规则：`REQ-NNN`
> 决策记录：复权方案选B（前复权+后复权各存一份），指数成分股权重要，正式版所有基础数据全部同步
> 开发策略：小步快跑，迭代推进

---

## 迭代规划

### Iteration 1（当前）— MVP：核心数据链路
**目标**：跑通"采集→存储→API查询"最小闭环
**需求**：REQ-001, REQ-003, REQ-005
**排期**：2周
**闭环标准**：`curl http://D:8000/v1/daily/000001.XSHE` 返回正确日线数据

### Iteration 2（待排期）— 基础数据扩展
**目标**：补充基础信息、分钟线、指数、市值、行业等
**需求**：REQ-002, REQ-004, REQ-006, REQ-007, REQ-009, REQ-011, REQ-012, REQ-013, REQ-014, REQ-015, REQ-016, REQ-021
**前提**：正式版账号到位

### Iteration 3（待排期）— 特色数据+监控
**目标**：接入特色数据、质量监控、应用对接
**需求**：REQ-008, REQ-010, REQ-017, REQ-018, REQ-019, REQ-020
**前提**：正式版账号到位，核心链路稳定

## Iteration 1 需求（已排期，开发中）

### REQ-001: 核心数据层部署
- **类型**: deploy
- **状态**: 🟢 已排期
- **优先级**: P0（阻塞后续所有开发）
- **迭代**: Iteration 1
- **描述**: 在服务器D部署ClickHouse主节点+Redis+FastAPI，服务器C部署ClickHouse从节点+Grafana
- **验收标准**:
  - [ ] D和C上Docker正常运行
  - [ ] ClickHouse主从复制正常
  - [ ] Redis可读写
  - [ ] FastAPI服务可访问
  - [ ] Grafana能查看监控面板
- **关联文档**: `docs/05-deployment.md`

---

### REQ-003: 日线行情同步（前复权 + 后复权）
- **类型**: data
- **状态**: 🟢 已排期
- **优先级**: P0
- **迭代**: Iteration 1
- **描述**: 同步沪深A股日线数据，前复权和后复权各存一份
- **API**: `get_price(fq='pre')`, `get_price(fq='post')`
- **字段**: open, high, low, close, volume, money, factor, high_limit, low_limit, avg, pre_close, paused
- **验收标准**:
  - [ ] `stock_daily_pre` 表（前复权）：全市场，账号可用范围内
  - [ ] `stock_daily_post` 表（后复权）：全市场，账号可用范围内
  - [ ] 每日收盘后自动同步当日数据
  - [ ] 与JQData源数据抽样比对一致
- **关联文档**: `docs/03-database-schema.md`

---

### REQ-005: REST API — 行情查询
- **类型**: feat
- **状态**: 🟢 已排期
- **优先级**: P0
- **迭代**: Iteration 1
- **描述**: 提供REST接口查询日线行情
- **验收标准**:
  - [ ] GET `/v1/daily/{code}` — 查询单股票日线（支持pre/post/none参数）
  - [ ] POST `/v1/daily/batch` — 批量查询日线
  - [ ] 支持时间范围筛选
  - [ ] 支持字段筛选
  - [ ] P95响应时间 < 200ms
  - [ ] 接口文档自动生成（Swagger/OpenAPI）
- **关联文档**: `docs/04-api-spec.md`
